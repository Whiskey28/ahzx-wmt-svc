# 安全、权限与数据治理（Architecture）

## 1. 权限模型

- **功能权限**：菜单 + 按钮 + API（与 `wmt-module-system` 一致）。
- **数据权限**：部门、自定义规则、仅本人等（`wmt-spring-boot-starter-biz-data-permission` 若已启用则对齐；否则在架构冻结前确认启用路径）。
- **角色模板建议**：`CUSTOMER`（仅小程序，无后台）、`MARKETING`、`RISK`、`POSTLOAN`、`ADMIN`、`AUDITOR`（只读）。

## 2. 多用户与公司级协作

- **组织树**：所有内部用户挂载部门；数据权限以部门为主轴扩展。
- **跨部门协查**：通过 **申请单授权** 或 **工单指派** 明确客体，避免全员可见。

## 3. 敏感数据分级（示例）


| 级别    | 示例             | 存储         | 展示       | 导出   |
| ----- | -------------- | ---------- | -------- | ---- |
| L3 极高 | 身份证、银行卡、征信原文   | 加密/分段；最小留存 | 强脱敏；二次校验 | 默认禁止 |
| L2 高  | 手机号、住址、额度、逾期信息 | 标准表；访问审计   | 掩码       | 审批流  |
| L1 中  | 进件状态、营销标签      | 普通         | 内部可见     | 受控   |


## 4. 小程序特有

- **会话**：短效 Access Token + Refresh；设备绑定可选。
- **防重放/nonce**：关键写接口限流 + 幂等键（如 `Idempotency-Key`）。
- **合规**：用户协议与隐私弹窗记录版本号与时间。

## 5. 贷后与催收

- **强审计**：工单每一步动作人、时间、渠道、结果码。
- **对接外部拨号/短信**：抽象 Provider，防止供应商锁定；日志不落完整被叫号码（按需掩码）。

## 6. 监管报送（分期）

- 单独 **报送任务表**：批次、状态、文件路径、错误摘要；支持重跑。
- 报送数据与业务库 **逻辑隔离**（视图/只读副本/ODS），避免在线库heavy scan。

## 7. 验收用例（安全）

- 横向：用户 A 不可读用户 B 的进件（同角色不同数据范围）。
- 纵向：小程序 Token 不可调 `/admin` 写接口。
- 导出：无审批时返回业务错误码，不静默失败。
