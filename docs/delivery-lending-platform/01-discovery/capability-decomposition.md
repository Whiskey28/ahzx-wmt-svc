# 能力拆解（Discovery）— 对接表格需求

以下将图片中的业务表述分解为 **可实现的业务能力块**，便于映射模块、API 与数据实体。

## 1. 微信小程序


| 表格能力         | 能力块 ID    | 说明                       |
| ------------ | --------- | ------------------------ |
| 客户注册、登录      | CAP-C-001 | 微信登录绑定、会话安全、可选实名前置       |
| 额度测算         | CAP-C-002 | 测算申请表单、策略引擎对接（可先规则/Mock） |
| 贷款申请         | CAP-C-003 | 申请单创建、版本化、与内部案件关联        |
| 信息授权、材料上传    | CAP-C-004 | 授权书记录、对象存储、合规留存          |
| 准入结果查询       | CAP-C-005 | 只读状态机视图                  |
| 营销/申请进度、授信额度 | CAP-C-006 | 时间线 + 额度展示（字段脱敏）         |


## 2. 数字化管理平台


| 表格能力      | 能力块 ID    | 说明                                    |
| --------- | --------- | ------------------------------------- |
| 客户端 + 管理端 | CAP-M-001 | 双壳或统一 SPA + 路由隔离；后端 `/admin` 分层       |
| 数据驾驶舱、可视化 | CAP-M-002 | 指标服务 + 缓存；图表数据源 API                   |
| 业务流程自动化   | CAP-M-003 | 与 BPM/状态机二选一或组合；第一期可用 **简化状态 + 审批留言** |
| 智能获客、精准营销 | CAP-M-004 | 线索、标签、活动（先做数据模型与最小闭环）                 |
| 风险全周期管理   | CAP-M-005 | 授信环节风控视图 + 贷后衔接                       |
| 监管报送      | CAP-M-006 | 报送任务、文件/报文导出接口（实现分期）                  |


## 3. 贷后管理平台（子域）


| 表格能力        | 能力块 ID    | 说明                |
| ----------- | --------- | ----------------- |
| 贷后监测        | CAP-P-001 | 监测任务、指标采集、账户/借据维度 |
| 风险预警        | CAP-P-002 | 规则配置、触发、分级处置      |
| 催收管理        | CAP-P-003 | 工单、动作模板、留痕        |
| 流程标准化与智能化运营 | CAP-P-004 | SLA、队列分派、自动化策略占位  |


## 依赖关系（高层）

- CAP-C-003/CAP-C-005 **依赖** 内部案件实体（与 CAP-M 进件一致）。
- CAP-P-*** **依赖** 借据/合同/还款计划等主数据（第一期可简化为核心表）。
- CAP-M-002 **依赖** 指标维表与聚合任务（可与CAP-P-001共享监测数据）。
