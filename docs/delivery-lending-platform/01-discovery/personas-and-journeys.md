# 角色与用户旅程（Discovery）

## 角色（Personas）

| 角色 | 主要端 | 核心诉求 |
|------|--------|----------|
| **借款客户** | 微信小程序 | 快速完成测算/申请；知悉进度与额度；材料少跑腿 |
| **客户经理 / 营销** | 管理端（客户端口径） | 获客线索、任务跟进、客户触达记录 |
| **信审 / 风控** | 管理端 | 准入与授信决策支持；查看风险信号与资料 |
| **贷后运营** | 管理端 | 监测指标、预警处置、催收工单与留痕 |
| **系统管理员** | 管理端 | 组织用户、角色权限、配置与审计 |
| **高管/管理** | 管理端驾驶舱 | 关键指标与趋势（先占位后丰富） |
| **合规/监管对接** | 管理端 + 批处理 | 报送出口、审计与追溯（分期） |

## 关键旅程（简版）

### 旅程 A：C 端申贷（ happy path ）

登录/绑定 → 实名与授权确认 → 填写测算/申请 → 上传材料 → 查询准入与额度 → 等待放款/签约（后续阶段）。

### 旅程 B：内部审批与授信

进件同步至内部 → 风控人员补资料要求 → 决策结论 → 授信结果回写 C 端可查询状态。

### 旅程 C：贷后预警与催收

定时监测/外部数据触达 → 触发预警规则 → 生成工单 → 催收动作记录 → 关闭/升级。

## 权限关注点

- **客户数据**：默认仅 **客户本人（小程序）** 与 **被授权的内部角色** 可见。
- **贷后/催收**：强审计；敏感操作需 **二次校验或审批**（规则在二期细化）。
