# 内蒙国产化适配范围与验收标准（TongWeb/TongRDS）

## 1. 当前框架定义（适配基线）

- 运行时：Java 17 + Spring Boot 3.5.x（Jakarta 命名空间）
- 数据访问：MyBatis-Plus + `BaseMapperX` + 动态数据源（master/slave）
- 连接池：Druid（监控 + 连接池参数）
- 接口规范：`CommonResult` / `PageParam` / `PageResult`
- 基础设施：Redis、Quartz（JDBC JobStore）、Actuator、日志体系
- 目标适配：TongWeb（应用容器）+ TongRDS（数据库及相关中间件链路）

## 2. 能力点清单

### 2.1 TongWeb 适配能力点

- TW-C1 启动兼容：应用在 TongWeb 冷启动/热重启稳定
- TW-C2 Web 行为一致：Filter/Interceptor/异常处理/编码行为一致
- TW-C3 鉴权会话一致：登录、鉴权、会话生命周期一致
- TW-C4 资源稳定：线程/连接资源无泄漏、无异常增长
- TW-C5 可观测可运维：日志、Actuator、Druid 监控可用
- TW-C6 部署回滚可执行：升级与回滚路径可操作可验证

### 2.2 TongRDS 适配能力点

- TR-C1 连接兼容：JDBC URL/驱动/TLS/字符集可用
- TR-C2 SQL 兼容：MyBatis-Plus 生成 SQL 可执行且结果一致
- TR-C3 事务一致：提交/回滚语义正确
- TR-C4 主从路由正确：读写分离符合设计
- TR-C5 稳定性可接受：连接池与性能无明显退化
- TR-C6 故障恢复可用：短时断连后可恢复

## 5. 通过标准

- P0 用例通过率必须为 100%
- 核心链路（启动、登录、核心 CRUD、分页）与基线一致
- 事务无异常语义漂移（提交/回滚正确）
- 稳态运行无连接池耗尽、无线程持续堆积
- 关键观测链路可用（health、应用日志、SQL 统计）
- 至少完成一次可执行回滚演练（建议）

## 6. 失败判定标准

- 任一 P0 用例失败
- 启动失败或关键 Bean 初始化失败
- 登录/鉴权核心链路不可用
- 数据一致性错误、事务失效、主从路由错误
- 稳态运行出现连接池打满且无法恢复
- 问题无法定位（缺少必要日志/监控证据）
