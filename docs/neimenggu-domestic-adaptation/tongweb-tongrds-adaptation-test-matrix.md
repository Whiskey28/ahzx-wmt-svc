# TongWeb/TongRDS 适配测试矩阵（可执行版）

## 1. 目标与适用范围

- 目标：验证当前框架在 `TongWeb + TongRDS` 组合下的功能一致性、稳定性与可运维性。
- 适用系统：`ahzx-wmt-svc`（Spring Boot 3.5.x / Java 17 / MyBatis-Plus / Druid / dynamic datasource）。
- 验证基线：与现有可运行环境（建议称为 E1）对比，目标环境称为 E2。

## 2. 执行约定

- 结果状态：`PASS` / `FAIL` / `BLOCKED`
- 当前状态：`已验证` / `待验证` / `环境受限`
- 优先级：
  - `P0`：阻断上线项（必须全通过）
  - `P1`：关键稳定性与兼容项（建议全通过）
  - `P2`：增强信心项（建议灰度前通过）
- 每条用例至少保留 1 项证据（日志、响应、截图、SQL结果）

## 3. 通用通过/失败标准

### 3.1 通过标准（总体验收）

- P0 用例通过率 = 100%
- 核心业务链路（登录、核心 CRUD、分页）结果与基线一致
- 事务语义正确（提交成功、回滚彻底）
- 稳态压测无连接池耗尽、无线程持续堆积
- 监控与日志可用于定位问题（health/日志/SQL统计可查）

### 3.2 失败判定（任一命中即不通过）

- 服务无法启动或关键 Bean 初始化失败
- 核心链路不可用、鉴权异常、响应结构不兼容
- 出现数据不一致、事务失效、主从路由错误
- 稳态运行出现连接池打满或错误率持续升高
