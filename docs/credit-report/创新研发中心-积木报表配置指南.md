# 创新研发中心 - 积木报表在线填报表单配置指南

## 一、业务需求分析

根据 Excel 模板，创新研发中心需要填报以下数据：

| 产品类型 | 产品名称示例 | 统计字段 |
|---------|------------|---------|
| 信用报告产品 | 企业信用报告、招投标 | 当年信用报告产品提供次数 |
| 信用分产品 | 信用分 | 当年信用分产品提供次数 |
| 反诈产品 | 反欺诈 | 当年反欺诈产品提供次数 |

**数据存储表**：`report_fill_product_stat`（1:N 明细表）

---

## 二、数据库表结构说明

### 2.1 主记录表（必配）

**表名**：`report_fill_basic_info`

| 字段 | 类型 | 说明 | 数据来源 |
|-----|------|------|---------|
| `id` | BIGINT | 主键（record_id） | 自增序列 |
| `period_id` | VARCHAR(16) | 填报周期（如 2026-01） | `${param.period}` 或表单输入 |
| `role_id` | VARCHAR(64) | 角色ID（创新研发中心） | `${param.roleId}` |
| `report_id` | VARCHAR(64) | 报表模板ID | `${param.reportId}` 或固定值 |
| `creator` | VARCHAR(64) | 创建者 | `${param.userId}` |
| `updater` | VARCHAR(64) | 更新者 | `${param.userId}` |
| `create_time` | TIMESTAMP | 创建时间 | 触发器自动填充 |
| `update_time` | TIMESTAMP | 更新时间 | 触发器自动填充 |
| `deleted` | int2 | 逻辑删除 | 默认 0 |

**唯一约束**：`(role_id, period_id, report_id)` - 确保同一角色同一周期同一模板只有一条主记录。

### 2.2 产品统计明细表（核心表）

**表名**：`report_fill_product_stat`

| 字段 | 类型 | 说明 | 数据来源 |
|-----|------|------|---------|
| `id` | BIGINT | 主键 | 自增序列 |
| `record_id` | BIGINT | 关联主记录ID | `report_fill_basic_info.id` |
| `product_type` | VARCHAR(64) | 产品类型（字典code） | 固定值或下拉选择 |
| `product_name` | VARCHAR(256) | 产品名称 | 表单输入 |
| `year_count` | NUMERIC(18, 0) | 当年提供次数 | 表单输入（数字） |
| `creator` | VARCHAR(64) | 创建者 | `${param.userId}` |
| `updater` | VARCHAR(64) | 更新者 | `${param.userId}` |
| `create_time` | TIMESTAMP | 创建时间 | 触发器自动填充 |
| `update_time` | TIMESTAMP | 更新时间 | 触发器自动填充 |
| `deleted` | int2 | 逻辑删除 | 默认 0 |

**产品类型字典建议**：
- `credit_report`：信用报告产品
- `credit_score`：信用分产品
- `anti_fraud`：反诈产品

## 三、积木报表配置步骤

### 3.1 准备阶段

1. **确认数据库表已创建**
   - 执行 `credit-report-2026-01-29.sql` 创建表结构
   - 执行 `credit-report-2026-01-29-triggers.sql` 创建触发器

2. **导入 Excel 模板**
   - 在积木报表中导入创新研发中心的 Excel 模板
   - 生成报表模板，记录模板ID（`report_id`）

3. **确定角色ID**
   - 确认“创新研发中心”对应的 `role_id` 值（如：`innovation_rd_center`）

### 3.2 创建在线填报表单

1. 在积木报表中选择对应模板，进入 **"填报设计"**
2. 新建在线填报表单，命名为：**"创新研发中心-产品统计填报"**
3. 记录该表单的 `report_id`（用于后续 iframe URL 传参）

### 3.3 数据集配置

#### 3.3.1 主记录数据集（必配）

**数据集名称**：`report_fill_basic_info`

**数据源类型**：关系表数据集

**表名**：`report_fill_basic_info`

**字段绑定配置**：

| 字段名 | 控件类型 | 默认值/绑定 | 是否隐藏 | 说明 |
|-------|---------|------------|---------|------|
| `id` | 单行文本 | 自动生成 | 是 | 主键，首次填报时自动生成 |
| `period_id` | 单行文本 | `${param.period}` | 否 | 填报周期，可用户输入或从参数获取 |
| `role_id` | 单行文本 | `${param.roleId}` | 是 | 角色ID，隐藏字段 |
| `report_id` | 单行文本 | `${param.reportId}` | 是 | 报表模板ID，隐藏字段 |
| `creator` | 单行文本 | `${param.userId}` | 是 | 创建者，隐藏字段 |
| `updater` | 单行文本 | `${param.userId}` | 是 | 更新者，隐藏字段 |

**关键配置**：
- 勾选“主键字段”：`id`
- 勾选“自动生成主键”：是
- 编辑模式查询条件：`role_id = ${param.roleId} AND period_id = ${param.period} AND report_id = ${param.reportId}`

#### 3.3.2 产品统计明细数据集（核心）

**数据集名称**：`report_fill_product_stat`

**数据源类型**：关系表数据集

**表名**：`report_fill_product_stat`

**字段绑定配置**：

| 字段名 | 控件类型 | 默认值/绑定 | 是否隐藏 | 说明 |
|-------|---------|------------|---------|------|
| `id` | 单行文本 | 自动生成 | 是 | 主键，自动生成 |
| `record_id` | 单行文本 | `report_fill_basic_info.id` | 是 | 关联主记录ID，隐藏字段 |
| `product_type` | 下拉选择 | 见下方说明 | 否 | 产品类型（字典） |
| `product_name` | 单行文本 | 用户输入 | 否 | 产品名称 |
| `year_count` | 数字 | 用户输入 | 否 | 当年提供次数（整数） |
| `creator` | 单行文本 | `${param.userId}` | 是 | 创建者，隐藏字段 |
| `updater` | 单行文本 | `${param.userId}` | 是 | 更新者，隐藏字段 |

**产品类型下拉选项配置**：

| 显示文本 | 值（code） |
|---------|-----------|
| 信用报告产品 | `credit_report` |
| 信用分产品 | `credit_score` |
| 反诈产品 | `anti_fraud` |

**关键配置**：
- 勾选“子表数据集”：是
- 关联主表：`report_fill_basic_info`
- 关联字段：`record_id = report_fill_basic_info.id`
- 编辑模式查询条件：`record_id = ${param.recordId}`（如果有 recordId 参数）

### 3.4 表单布局设计

#### 3.4.1 表头区域

```
┌─────────────────────────────────────────┐
│  填报月份：[period_id 控件]              │
└─────────────────────────────────────────┘
```

- **填报月份**：绑定到 `report_fill_basic_info.period_id`
- 控件类型：单行文本或日期选择器
- 默认值：`${param.period}`（如：2026-01）

#### 3.4.2 产品统计明细区域（子表）

**使用积木报表的“子表控件”或“多行区域”功能**

**表格布局建议**：

```
┌──────────────────────────────────────────────────────────────┐
│ 提供的征信产品(服务)的名称及次数                              │
├──────────────┬──────────────────┬───────────────────────────┤
│ 产品类型      │ 产品名称         │ 当年提供次数              │
├──────────────┼──────────────────┼───────────────────────────┤
│ [下拉选择]    │ [单行文本]       │ [数字]                    │
│ [下拉选择]    │ [单行文本]       │ [数字]                    │
│ [下拉选择]    │ [单行文本]       │ [数字]                    │
│ ...          │ ...              │ ...                       │
└──────────────┴──────────────────┴───────────────────────────┘
```

**单元格绑定**：

| 列标题 | 单元格绑定 | 控件类型 | 必填 |
|-------|-----------|---------|------|
| 产品类型 | `report_fill_product_stat.product_type` | 下拉选择 | 是 |
| 产品名称 | `report_fill_product_stat.product_name` | 单行文本 | 是 |
| 当年提供次数 | `report_fill_product_stat.year_count` | 数字 | 是 |

**子表配置要点**：
- 启用“允许新增行”：是
- 启用“允许删除行”：是
- 最小行数：0（允许空表）
- 最大行数：建议 100（根据实际需求调整）

### 3.5 隐藏字段配置

在表单中创建隐藏单元格，用于存储系统参数：

| 字段路径 | 默认值 | 说明 |
|---------|-------|------|
| `report_fill_basic_info.role_id` | `${param.roleId}` | 角色ID（创新研发中心） |
| `report_fill_basic_info.report_id` | `${param.reportId}` | 报表模板ID |
| `report_fill_basic_info.creator` | `${param.userId}` | 创建者 |
| `report_fill_basic_info.updater` | `${param.userId}` | 更新者 |
| `report_fill_product_stat.record_id` | `report_fill_basic_info.id` | 关联主记录ID |
| `report_fill_product_stat.creator` | `${param.userId}` | 创建者 |
| `report_fill_product_stat.updater` | `${param.userId}` | 更新者 |

**配置方法**：
1. 在表格右侧属性面板中，选择对应单元格
2. 控件类型：单行文本
3. 绑定字段：选择上述字段路径
4. 默认值：填入对应的 `${param.xxx}` 表达式
5. 勾选“只读”和“隐藏”

## 四、iframe URL 参数配置

### 4.1 URL 参数清单

在业务系统中嵌入积木报表 iframe 时，需要传递以下参数：

| 参数名 | 说明 | 示例值 | 必填 |
|-------|------|--------|------|
| `roleId` | 角色ID（创新研发中心） | `innovation_rd_center` | 是 |
| `reportId` | 报表模板ID | `jimu_report.id` 的值 | 是 |
| `period` | 填报周期 | `2026-01` | 是 |
| `userId` | 当前登录用户ID | `123` | 是 |
| `recordId` | 已有记录ID（编辑模式） | `456` | 否 |

### 4.2 URL 示例

**新建模式**：
```
http://jimu-report-host/fill?roleId=innovation_rd_center&reportId=xxx&period=2026-01&userId=123
```

**编辑模式**：
```
http://jimu-report-host/fill?roleId=innovation_rd_center&reportId=xxx&period=2026-01&userId=123&recordId=456
```

## 五、数据验证规则

### 5.1 字段级验证

| 字段 | 验证规则 | 错误提示 |
|-----|---------|---------|
| `period_id` | 必填，格式：YYYY-MM | "请选择填报月份" |
| `product_type` | 必填，值必须在字典中 | "请选择产品类型" |
| `product_name` | 必填，长度 ≤ 256 | "请输入产品名称" |
| `year_count` | 必填，整数 ≥ 0 | "请输入有效的提供次数（≥0）" |

### 5.2 业务级验证

- 同一 `record_id` 下，`product_type` 不能重复（可选，根据业务需求）
- `year_count` 必须为非负整数

## 七、常见问题排查

### 7.1 数据未保存

**可能原因**：
- 主记录数据集未正确配置主键
- 子表数据集未正确关联主表
- 隐藏字段未正确绑定 `${param.xxx}`

**排查步骤**：
1. 检查积木报表的“数据集管理”，确认数据集配置正确
2. 检查表单单元格绑定，确认字段路径正确
3. 检查 iframe URL 参数，确认所有必填参数已传递

### 7.2 明细数据未关联

**可能原因**：
- `record_id` 字段未正确绑定到主记录 `id`
- 子表数据集关联配置错误

**排查步骤**：
1. 检查 `report_fill_product_stat.record_id` 是否绑定为 `report_fill_basic_info.id`
2. 检查子表数据集的“关联主表”配置

### 7.3 审计字段未填充

**可能原因**：
- 触发器未创建或未生效
- `creator`、`updater` 未从 `${param.userId}` 获取

**排查步骤**：
1. 检查数据库触发器：`SELECT * FROM pg_trigger WHERE tgname LIKE '%report_fill%';`
2. 检查表单隐藏字段，确认 `creator`、`updater` 绑定为 `${param.userId}`

## 八、扩展建议

### 8.1 产品类型字典化

建议在系统字典表中创建产品类型字典：

**字典类型**：`credit_product_type`

**字典数据**：
- `credit_report`：信用报告产品
- `credit_score`：信用分产品
- `anti_fraud`：反诈产品

在积木报表中，产品类型下拉框可配置为从字典表读取。

### 8.2 数据汇总统计

后续如需按周期、按角色汇总产品提供次数，可使用 SQL：

```sql
SELECT 
    r.period_id,
    r.role_id,
    p.product_type,
    SUM(p.year_count) AS total_count
FROM report_fill_basic_info r
INNER JOIN report_fill_product_stat p ON r.id = p.record_id
WHERE r.deleted = 0 AND p.deleted = 0
GROUP BY r.period_id, r.role_id, p.product_type;
```
