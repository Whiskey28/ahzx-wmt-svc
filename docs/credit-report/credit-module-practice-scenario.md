# 征信模块实践场景示例

## 📋 场景说明

本文档通过一个完整的实践场景，展示管理员和表单专员如何从零开始使用征信模块。

**场景背景：**
- 公司有3个部门：财务部、业务部、技术部
- 需要按月报送征信数据
- 2024年1月需要完成首次报送

---

## 场景一：管理员首次配置（2024年1月1日）

### 步骤1：管理员登录系统

**操作：**
1. 打开系统登录页面
2. 输入管理员账号：`admin`
3. 输入密码：`********`
4. 点击"登录"

**预期结果：**
- 登录成功，进入系统主界面
- 可以看到"征信管理"菜单

---

### 步骤2：配置通用字段

**目标：** 创建所有部门都需要填写的字段

**操作：**

#### 2.1 创建"资产总额"字段

1. 点击菜单：**"征信管理"** → **"字段配置"**
2. 点击右上角 **"新增"** 按钮
3. 填写字段信息：
   ```
   部门编号：0
   字段编码：total_assets
   字段名称：资产总额
   字段类型：DECIMAL
   是否必填：是
   默认值：0
   校验规则：
     - 最小值：0
     - 最大值：999999999.99
     - 小数位数：2
   显示顺序：1
   状态：启用
   ```
4. 点击 **"确定"** 保存

**预期结果：**
- 字段创建成功，出现在字段列表中
- 字段编码为 `total_assets`，部门编号为 `0`

#### 2.2 创建"负债总额"字段

1. 点击 **"新增"** 按钮
2. 填写字段信息：
   ```
   部门编号：0
   字段编码：total_liabilities
   字段名称：负债总额
   字段类型：DECIMAL
   是否必填：是
   默认值：0
   校验规则：
     - 最小值：0
     - 最大值：999999999.99
     - 小数位数：2
   显示顺序：2
   状态：启用
   ```
3. 点击 **"确定"** 保存

#### 2.3 创建"净资产"字段

1. 点击 **"新增"** 按钮
2. 填写字段信息：
   ```
   部门编号：0
   字段编码：net_assets
   字段名称：净资产
   字段类型：DECIMAL
   是否必填：是
   默认值：0
   校验规则：
     - 最小值：-999999999.99
     - 最大值：999999999.99
     - 小数位数：2
   显示顺序：3
   状态：启用
   ```
3. 点击 **"确定"** 保存

**当前状态：**
- 已创建3个通用字段
- 所有部门在填写表单时都会看到这3个字段

---

### 步骤3：配置部门特有字段

**目标：** 为财务部创建特有字段"预算执行率"

**操作：**

1. 点击 **"新增"** 按钮
2. 填写字段信息：
   ```
   部门编号：财务部（假设ID=2）
   字段编码：budget_execution_rate
   字段名称：预算执行率
   字段类型：DECIMAL
   是否必填：是
   默认值：0
   校验规则：
     - 最小值：0
     - 最大值：100
     - 小数位数：2
   显示顺序：10
   状态：启用
   ```
3. 点击 **"确定"** 保存

**当前状态：**
- 财务部在填写表单时会看到4个字段（3个通用字段 + 1个部门字段）
- 业务部和技术部只看到3个通用字段

---

### 步骤4：配置计算规则

**目标：** 配置月报汇总计算规则

**操作：**

#### 4.1 创建"总资产"计算规则

1. 点击菜单：**"征信管理"** → **"计算规则配置"**
2. 点击右上角 **"新增"** 按钮
3. 填写计算规则信息：
   ```
   目标字段编码：total_assets_sum
   报表类型：MONTHLY
   规则类型：SUM
   计算表达式：#deptFormData.values().![get('total_assets')].sum()
   数据来源：{"type": "FORM_DATA", "fields": ["total_assets"]}
   计算顺序：1
   规则描述：计算各部门资产总额之和
   状态：启用
   ```
4. 点击 **"确定"** 保存

#### 4.2 创建"总负债"计算规则

1. 点击 **"新增"** 按钮
2. 填写计算规则信息：
   ```
   目标字段编码：total_liabilities_sum
   报表类型：MONTHLY
   规则类型：SUM
   计算表达式：#deptFormData.values().![get('total_liabilities')].sum()
   数据来源：{"type": "FORM_DATA", "fields": ["total_liabilities"]}
   计算顺序：2
   规则描述：计算各部门负债总额之和
   状态：启用
   ```
3. 点击 **"确定"** 保存

#### 4.3 创建"总净资产"计算规则

1. 点击 **"新增"** 按钮
2. 填写计算规则信息：
   ```
   目标字段编码：net_assets_sum
   报表类型：MONTHLY
   规则类型：FORMULA
   计算表达式：#total_assets_sum - #total_liabilities_sum
   数据来源：{"type": "CALCULATED", "fields": ["total_assets_sum", "total_liabilities_sum"]}
   计算顺序：3
   规则描述：总净资产 = 总资产 - 总负债
   状态：启用
   ```
3. 点击 **"确定"** 保存

**当前状态：**
- 已创建3个计算规则
- 月报生成时会自动计算这3个汇总字段

---

### 步骤5：通知表单专员开始填写

**操作：**
1. 管理员通过邮件或系统消息通知3个部门的表单专员
2. 通知内容：
   ```
   主题：2024年1月征信数据填报通知
   
   各位同事：
   
   请于2024年1月25日前完成2024年1月的征信数据填报。
   
   填报步骤：
   1. 登录系统
   2. 进入"征信管理" → "表单填报"
   3. 选择报送周期：2024-01
   4. 选择报表类型：月报
   5. 填写表单数据
   6. 保存草稿（可多次保存）
   7. 提交表单
   
   如有问题，请联系管理员。
   ```

**当前状态：**
- 配置工作已完成
- 等待表单专员填写表单

---

## 场景二：表单专员填写表单（2024年1月10日）

### 步骤1：财务部专员登录系统

**操作：**
1. 打开系统登录页面
2. 输入账号：`finance_user`
3. 输入密码：`********`
4. 点击"登录"

**预期结果：**
- 登录成功，进入系统主界面
- 可以看到"征信管理"菜单

---

### 步骤2：进入表单填报页面

**操作：**
1. 点击菜单：**"征信管理"** → **"表单填报"**
2. 在页面顶部选择：
   - **报送周期**：`2024-01`
   - **报表类型**：`MONTHLY`（月报）
3. 点击 **"开始填报"** 按钮

**预期结果：**
- 页面显示表单，包含4个字段：
  - 资产总额（必填）
  - 负债总额（必填）
  - 净资产（必填）
  - 预算执行率（必填，财务部特有字段）

---

### 步骤3：填写表单数据

**操作：**

#### 3.1 填写"资产总额"

1. 找到"资产总额"字段（显示顺序为1，在最上方）
2. 在输入框中输入：`5000000.00`
3. 鼠标移出输入框，系统自动校验
4. 校验通过，无错误提示

#### 3.2 填写"负债总额"

1. 找到"负债总额"字段
2. 在输入框中输入：`2000000.00`
3. 鼠标移出输入框，系统自动校验
4. 校验通过，无错误提示

#### 3.3 填写"净资产"

**注意：** 净资产 = 资产总额 - 负债总额 = 5000000.00 - 2000000.00 = 3000000.00

1. 找到"净资产"字段
2. 在输入框中输入：`3000000.00`
3. 鼠标移出输入框，系统自动校验
4. 校验通过，无错误提示

#### 3.4 填写"预算执行率"

1. 找到"预算执行率"字段
2. 在输入框中输入：`85.50`
3. 鼠标移出输入框，系统自动校验
4. 校验通过，无错误提示

**当前状态：**
- 所有字段都已填写
- 所有字段值都符合校验规则

---

### 步骤4：保存草稿

**操作：**
1. 点击页面底部的 **"保存草稿"** 按钮
2. 系统提示："保存成功"
3. 页面显示表单ID（例如：`1001`）

**预期结果：**
- 表单数据已保存
- 可以随时继续编辑

---

### 步骤5：检查数据并提交

**操作：**
1. 再次检查所有字段值是否正确
2. 确认无误后，点击页面底部的 **"提交"** 按钮
3. 系统进行最终校验
4. 校验通过，系统提示："提交成功"
5. 表单状态变为"已提交"

**预期结果：**
- 表单提交成功
- 表单状态为"已提交"
- 表单无法再修改

---

### 步骤6：业务部和技术部专员填写表单

**操作：**
- 业务部专员和技术部专员按照相同步骤填写表单
- 他们只看到3个通用字段（没有"预算执行率"字段）

**业务部数据示例：**
```
资产总额：3000000.00
负债总额：1000000.00
净资产：2000000.00
```

**技术部数据示例：**
```
资产总额：2000000.00
负债总额：800000.00
净资产：1200000.00
```

**当前状态：**
- 财务部表单已提交
- 业务部表单已提交
- 技术部表单已提交
- 所有部门表单都已提交，可以生成汇总报表

---

## 场景三：管理员生成汇总报表（2024年1月26日）

### 步骤1：检查表单提交情况

**操作：**
1. 点击菜单：**"征信管理"** → **"表单数据"**
2. 筛选条件：
   - **报送周期**：`2024-01`
   - **报表类型**：`MONTHLY`
   - **状态**：`已提交`
3. 点击 **"查询"** 按钮

**预期结果：**
- 显示3条表单数据：
  - 财务部：已提交
  - 业务部：已提交
  - 技术部：已提交
- 所有部门表单都已提交

---

### 步骤2：生成汇总报表

**操作：**
1. 点击菜单：**"征信管理"** → **"汇总报表"**
2. 点击右上角 **"生成报表"** 按钮
3. 填写生成参数：
   - **报送周期**：`2024-01`
   - **报表类型**：`MONTHLY`（月报）
4. 点击 **"确定"** 按钮
5. 系统开始计算，页面显示"计算中"状态
6. 等待几秒钟，状态变为"已完成"

**预期结果：**
- 报表生成成功
- 报表状态为"已完成"
- 报表ID为 `2001`（示例）

---

### 步骤3：查看汇总报表数据

**操作：**
1. 在汇总报表列表中，找到刚生成的报表（ID=2001）
2. 点击该行的 **"查看"** 按钮
3. 查看报表数据

**预期结果：**
报表数据如下：
```
总资产（total_assets_sum）：10000000.00
  = 财务部 5000000.00 + 业务部 3000000.00 + 技术部 2000000.00

总负债（total_liabilities_sum）：3800000.00
  = 财务部 2000000.00 + 业务部 1000000.00 + 技术部 800000.00

总净资产（net_assets_sum）：6200000.00
  = 总资产 10000000.00 - 总负债 3800000.00
```

**验证计算：**
- ✅ 总资产 = 5000000 + 3000000 + 2000000 = 10000000 ✓
- ✅ 总负债 = 2000000 + 1000000 + 800000 = 3800000 ✓
- ✅ 总净资产 = 10000000 - 3800000 = 6200000 ✓

---

### 步骤4：导出Excel

**操作：**
1. 在汇总报表列表中，找到报表（ID=2001）
2. 点击该行的 **"导出Excel"** 按钮
3. 系统自动下载Excel文件：`征信汇总报表.xls`
4. 打开Excel文件查看数据

**预期结果：**
- Excel文件包含报表的所有数据
- 表头使用字段名称（不是字段编码）
- 数据格式正确

## 场景五：修改表单数据（异常情况）

### 场景：财务部发现数据错误，需要修改

**前提：** 表单已提交，但汇总报表尚未生成

**操作：**

#### 方案1：管理员撤销提交（如果系统支持）

1. 管理员登录系统
2. 进入"表单数据"页面
3. 找到财务部的表单
4. 点击"撤销提交"按钮（如果系统支持）
5. 表单状态变为"草稿"
6. 通知财务部专员修改数据
7. 财务部专员修改后重新提交

#### 方案2：管理员直接修改（如果系统支持）

1. 管理员登录系统
2. 进入"表单数据"页面
3. 找到财务部的表单
4. 点击"编辑"按钮
5. 修改数据
6. 保存并重新提交

#### 方案3：删除后重新创建（如果系统不支持撤销）

1. 管理员删除已提交的表单
2. 财务部专员重新填写表单
3. 重新提交

**注意事项：**
- 如果汇总报表已生成，需要重新生成报表
- 修改数据后，审计日志会记录变更记录

## 总结

### 完整流程总结

```
管理员配置阶段：
1. 登录系统
2. 配置通用字段（所有部门共享）
3. 配置部门字段（部门特有）
4. 配置计算规则（月报、季报）
5. 通知表单专员

表单专员填写阶段：
1. 登录系统
2. 选择报送周期
3. 填写表单数据
4. 保存草稿（可多次）
5. 提交表单

管理员汇总阶段：
1. 检查所有表单是否已提交
2. 生成汇总报表
3. 查看报表数据
4. 导出Excel
5. 查看审计日志
```

### 关键时间节点

| 时间 | 操作 | 责任人 |
|------|------|--------|
| 1月1日 | 配置字段和计算规则 | 管理员 |
| 1月10日 | 财务部提交表单 | 财务部专员 |
| 1月15日 | 业务部提交表单 | 业务部专员 |
| 1月20日 | 技术部提交表单 | 技术部专员 |
| 1月26日 | 生成汇总报表 | 管理员 |
| 4月1日 | 生成Q1季报 | 管理员 |

### 注意事项

1. **字段配置**：首次使用前必须配置字段和计算规则
2. **表单提交**：所有部门表单提交后才能生成汇总报表
3. **数据准确性**：提交前请仔细核对数据，提交后无法修改
4. **时间节点**：建议在每月25日前完成表单提交，26日生成报表
5. **审计日志**：所有操作都会记录在审计日志中，可追溯
