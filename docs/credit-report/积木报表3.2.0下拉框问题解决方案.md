# 积木报表3.2.0 下拉框首次提交获取不到值的问题解决方案

## 问题描述

在积木报表3.2.0版本中，配置下拉框进行填报时：
- ❌ **第一次提交**：INSERT 语句获取不到下拉框的值
- ✅ **后续编辑**：UPDATE 语句可以正常获取下拉框的值

## 问题原因分析

这是积木报表3.2.0版本的一个已知问题，主要原因包括：

1. **下拉框数据源初始化时机问题**：首次提交时，下拉框的数据源可能还未完全加载
2. **字段绑定时机问题**：INSERT 操作时，下拉框的值可能还未正确绑定到数据集字段
3. **数据集关联问题**：子表数据集的下拉框字段在首次插入时，可能因为主表ID还未生成而无法正确关联

## 解决方案

### 方案一：使用静态字典下拉框（推荐）

**适用场景**：下拉框选项固定且数量较少（如产品类型、行业分类等）

**配置步骤**：

1. **在积木报表中配置下拉框**
   - 控件类型：选择"下拉框"
   - 数据字典类型：选择"静态数据"
   - 静态数据配置：
     ```
     信用报告产品,credit_report
     信用分产品,credit_score
     反诈产品,anti_fraud
     ```
   - **关键配置**：
     - 显示字段：第一列（信用报告产品）
     - 值字段：第二列（credit_report）
     - **默认值**：留空或设置默认选项

2. **字段绑定**
   - 绑定字段：`report_fill_product_stat.product_type`
   - 字段类型：`VARCHAR(64)`
   - **重要**：确保下拉框的"值字段"与数据库字段类型匹配

3. **初始化脚本（可选）**
   - 在数据集配置中，添加初始化脚本：
   ```javascript
   // 确保下拉框值在提交前已设置
   if (this.product_type == null || this.product_type == '') {
       // 如果为空，可以设置默认值或抛出错误
   }
   ```

### 方案二：使用SQL数据集下拉框（动态数据）

**适用场景**：下拉框选项需要从数据库动态加载

**配置步骤**：

1. **创建下拉框数据源数据集**
   - 数据集名称：`dict_product_type`
   - 数据源类型：SQL数据集
   - SQL语句：
   ```sql
   SELECT 'credit_report' AS value, '信用报告产品' AS label
   UNION ALL
   SELECT 'credit_score' AS value, '信用分产品' AS label
   UNION ALL
   SELECT 'anti_fraud' AS value, '反诈产品' AS label
   ```

2. **配置下拉框控件**
   - 控件类型：下拉框
   - 数据字典类型：选择"SQL数据集"
   - 数据源：选择 `dict_product_type`
   - 显示字段：`label`
   - 值字段：`value`

3. **字段绑定**
   - 绑定字段：`report_fill_product_stat.product_type`
   - **关键**：确保下拉框的"值字段"与数据库字段类型匹配

4. **添加初始化脚本**
   ```javascript
   // 在数据集初始化时，确保下拉框数据已加载
   // 这需要在数据集配置的"初始化脚本"中设置
   ```

### 方案三：使用文本输入框 + 数据验证（临时方案）

**适用场景**：下拉框问题无法解决时的临时方案

**配置步骤**：

1. **将下拉框改为文本输入框**
   - 控件类型：单行文本
   - 绑定字段：`report_fill_product_stat.product_type`

2. **添加数据验证**
   - 在数据集配置中添加验证规则：
   ```javascript
   // 验证 product_type 是否为有效值
   var validTypes = ['credit_report', 'credit_score', 'anti_fraud'];
   if (validTypes.indexOf(this.product_type) === -1) {
       throw new Error('产品类型无效，请输入：credit_report、credit_score 或 anti_fraud');
   }
   ```

3. **前端提示**
   - 在表单中添加提示文本，告知用户可输入的值

### 方案四：使用积木报表的"字典管理"功能

**适用场景**：需要统一管理字典数据

**配置步骤**：

1. **在积木报表中创建字典**
   - 进入"系统管理" → "字典管理"
   - 创建字典类型：`credit_product_type`
   - 添加字典项：
     - `credit_report`：信用报告产品
     - `credit_score`：信用分产品
     - `anti_fraud`：反诈产品

2. **配置下拉框**
   - 控件类型：下拉框
   - 数据字典类型：选择"系统字典"
   - 字典类型：选择 `credit_product_type`
   - 显示字段：`label`
   - 值字段：`value`

3. **字段绑定**
   - 绑定字段：`report_fill_product_stat.product_type`

## 推荐配置检查清单

在配置下拉框时，请确保以下配置正确：

- [ ] **下拉框数据源已正确配置**（静态数据/SQL数据集/系统字典）
- [ ] **值字段与数据库字段类型匹配**（VARCHAR(64)）
- [ ] **字段绑定路径正确**（`report_fill_product_stat.product_type`）
- [ ] **子表数据集已正确关联主表**（`record_id = report_fill_basic_info.id`）
- [ ] **主表ID在子表插入前已生成**（确保主表先插入）
- [ ] **初始化脚本已配置**（如有必要）
- [ ] **测试首次提交和编辑更新**（两种场景都要测试）

## 如果以上方案都无法解决

如果尝试了以上所有方案，下拉框在首次提交时仍然无法获取值，建议采用**扁平化表结构设计**（见下文）。
