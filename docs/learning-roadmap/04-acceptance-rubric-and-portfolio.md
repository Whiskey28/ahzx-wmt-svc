# 各阶段量化验收建议与作品集清单

用于自学或与导师对齐；阈值可按团队标准调整。

## 阶段一：中级 → 钉在代码库上

| 维度 | 建议标准 |
|------|----------|
| 启动与模块 | 能画一张图：Server → 依赖模块 → 所选业务包；说明 `admin-api` 前缀来源 |
| 业务链路 | 至少 **1 条** 端到端链路（Controller→Service→Mapper），含异常路径口述 |
| 联调 | 浏览器 Network 能对应到后端路径与 `CommonResult` 结构 |
| 质量 | 至少 **1 个** 合并进主干的测试类（WebMvcTest 或单测），与项目规范一致 |

## 阶段二：技术专家向

| 维度 | 建议标准 |
|------|----------|
| 性能/SQL | **1 次** 有前后对比的优化或分析（截图/数字：优化前 RT、优化后 RT） |
| 事务/一致性 | 能解释所选模块中 **1 处** `@Transactional` 边界或幂等设计 |
| 演练 | **1 份** 完整填写的故障/性能复盘（见 `02-phase2` 附录模板） |
| 可观测性能 | 能指出本项目监控入口与 **2 个** 核心指标含义 |

## 阶段三：全链路

| 维度 | 建议标准 |
|------|----------|
| 产品 | **1 份** PRD（`03` 模板 A），含可量化成功标准 |
| 交付 | **1 份** 技术方案 + **1 份** 上线记录 |
| 运营 | **1 份** 上线后指标复盘（`03` 模板 D） |

---

## 作品集（脱敏）建议清单

适合面试或晋升材料，各保留 **1 例** 即可：

1. **架构图**：C4 中 Container/Component 一页（本仓库 + Nginx + DB + 前端）。
2. **PRD + 方案**：同一件事的「业务语言」与「技术语言」对照。
3. **测试证据**：关键接口的 WebMvcTest 或集成测试片段说明。
4. **上线与复盘**：Runbook + 指标前后对比（数字可模糊化）。
5. **个人总结**：3 分钟口述稿——「我解决了什么问题、如何验证、如何防止再犯」。
