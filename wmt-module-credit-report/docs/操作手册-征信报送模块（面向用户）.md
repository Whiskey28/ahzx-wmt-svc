# 征信报送模块（credit-report）操作手册（面向用户）

> 你不需要理解接口、数据库、报表引擎。你只要按本手册完成：**选周期 → 打开你负责的表单 → 把每个字段按口径填对 → 保存**。

## 1. 你在做什么（用一句话）

每个月（`YYYY-MM`）各部门把本部门负责的“征信相关指标”填入积木表单，系统自动汇总生成征信报送报表。

## 2. 你需要先确认的 3 件事

- **你的填报周期**：只能填 `YYYY-MM`，例如 `2026-01`
- **你的部门角色**：系统里对应“XX部填报人”（例如：财务部填报人、数据管理中心填报人等）
- **你负责的表单**：不同部门看到/填写的表单模板可能不同（由管理员配置）

## 3. 通用操作：从系统进入表单并保存

### 3.1 选择周期（periodId）

- 只填：`YYYY-MM`
- 例：`2026-01`
- 不要填：`2026/01`、`2026-1`、`2026-01-15`

### 3.2 选择模板并进入表单

典型步骤：

1. 进入【报表填报】页面
2. 在左侧/上方选择【分类】
3. 在模板列表中选择你要填的【在线填报表单】
4. 点击进入表单【编辑】页

### 3.3 保存与自查

- **保存**：填写完后点击【保存】（或【保存草稿】）
- **自查**：保存后刷新页面确认数据未丢失；必要时返回“填报记录”列表确认该周期下已存在记录

> 本业务 **没有审批流**，保存即代表该周期该模板下的数据会参与系统统计（按口径聚合）。

## 4. 表单字段怎么填（按页面/模块）

以下字段说明来自你们系统已落库的填报块（字段名仅用于解释口径，页面上看到的是中文标签）。

### 4.1 企业基础信息（企业概况）

填写要点：以企业证照/备案信息为准，时间按证照日期，金额注意单位。

- **企业名称**：营业执照全称
- **统一社会信用代码**：18 位信用代码
- **注册地/地址**：营业执照注册地址
- **法定代表人**：证照登记法定代表人
- **注册时间**：企业注册成立日期（按证照/登记机关）
- **经营场所**：实际经营地址（如与注册地址不同可据实填）
- **备案编号**：如有监管备案/资质备案编号则填写，没有可留空
- **备案时间**：对应备案编号的日期
- **备案分支行/机构**：备案/登记受理机构名称
- **注册资本（万元）**：单位是“万元人民币”
- **实缴资本（万元）**：单位是“万元人民币”
- **信息系统等级保护评定安全等级**：如“二级/三级”等（按已评定结论）
- **服务网络形式**：按你们内部口径选择（如“直营网点/线上为主/混合”等）
- **分支机构数量**：按本周期实际数量
- **控股股东**：控股股东名称
- **控股股东股份占比（%）**：例如“51%”
- **控股股东类型**：如“国有/民营/混合”等（按你们口径）
- **联系人**：本次填报对接人
- **联系电话**：联系人电话
- **联系邮箱**：联系人邮箱

常见错误：

- 金额单位填错（把“元”当成“万元”）
- 占比未带百分号或填成小数（建议按页面要求填写）

### 4.2 经营情况（财务口径）——通常由财务部填写

填写要点：金额单位为“元”，建议与财务报表口径一致（含/不含税以你们内部口径为准）。

- **资产（元）**
- **负债（元）**
- **本年度总收入（元）**
- **本年度企业征信业务收入（元）**
- **本年度净利润（元）**

自查建议：

- 征信业务收入 ≤ 总收入
- 资产、负债为期末口径还是平均口径：按你们模板说明一致填写

### 4.3 经营情况（人员口径）——通常由综合/人事口径填写

- **员工总人数**：本周期统计口径下的员工总数
- **从事征信业务员工数**：参与征信相关业务的人数

自查建议：

- 征信业务员工数 ≤ 员工总人数

### 4.4 经营情况（信用体系建设口径）——通常由相关业务部门填写

- **企业参与行业、地方信用体系建设项目数**：本周期/本年度按模板要求统计
- **企业参与行业、地方信用体系建设收入（元）**

### 4.5 信息采集情况（信息收录）

> 该块字段较多，页面上一般会按条目分组（企业类、自然人类、渠道类）。以下为当前系统已配置字段。

企业类（户数）：

- **收录企业（含个体工商户、小微企业）总户数**
- **收录借贷信息的企业户数**
- **收录商业交易信息的企业户数**
- **收录财务信息的企业户数**
- **收录税务信息的企业户数**
- **收录进出口信息的企业户数**
- **收录水电气等公用事业信息的企业户数**
- **收录场内企业户数**
- **收录场内小微企业总户数**
- **收录个体工商户总户数**

自然人/关键人：

- **收录自然人股东信息：公司董事、监事、高管人员人数**
- **收录关键人员人数**（如页面存在该项，以模板口径为准）

自查建议：

- 各分类户数不一定相加等于“总户数”（取决于口径是否互斥）；如模板要求“去重”或“可重复”，按模板说明。

### 4.6 异议/投诉处理与信息安全情况

投诉与异议：

- **企业投诉处理次数**
- **异议受理次数**
- **异议核实后准确的数量**
- **异议核实后更正或删除的数量**
- **异议核实无法确认的数量**

信息安全：

- **发生信息安全事件数**
- **发生信息泄露事件数**
- **信息泄露的企业数**
- **信息泄露的董事、监事、高级管理人员数**

自查建议：

- 异议“更正或删除数量”一般 ≤ “异议受理次数”
- 如某项为 0，建议填 0（不要留空，除非页面允许且明确表示“无数据”）

### 4.7 信息来源情况（按行业）——通常由数据管理中心填写

该块通常按行业逐行填写两列：

- **信息提供者机构总累计数**
- **当前提供服务的信息提供者机构数**

填写要点：

- 行业行顺序固定（由系统字典控制），按每行行业分别填写
- “当前提供服务数”应 ≤ “总累计数”

### 4.8 产品与服务提供情况（按行业）——通常由相关部门联合口径

该块通常按行业逐行填写：

- **当年提供产品（服务）次数**

填写要点：

- 行业行顺序固定，逐行填写次数（整数）
- 若本行业没有发生，填 0

### 4.9 提供的征信产品（服务）次数（按产品类别）

系统当前按固定三类统计（页面一般是 3 行或 3 个输入框）：

- **企业信用报告/招投标报告：当年提供次数**
- **信用分：当年提供次数**
- **反欺诈：当年提供次数**

重要说明（业务规则）：

- 系统在“部分汇总口径”中存在 **汇总后 + 3000 × 月数** 的规则（例如 1 月加 3000，2 月加 6000……），请以报表展示为准。

## 5. 信息使用者机构维护（如你有该权限）

这部分一般由管理员/数据维护人员操作，用于报表中的下拉树、以及“按行业统计信息使用者机构数量”。

### 5.1 维护“非政府机构明细”

你通常需要填写：

- **机构名称**：`orgName`
- **行业分类**：`industryCode`（从下拉选择）
- **是否当前提供服务**：`isCurrentService`（是/否）
- **排序号**：`sortNo`（数字越小越靠前）

### 5.2 维护“政府机构明细”

你通常需要填写：

- **政府机构名称**：`govOrgName`
- **是否当前提供服务**：`isCurrentService`
- **排序号**：`sortNo`

## 6. 常见问题（用户最常问）

### 6.1 我填完了，报表里没变化？

可能原因：

- 你填的周期不是 `YYYY-MM`（例如多了日期）
- 你填的是别的模板（确认模板名称与周期）
- 报表统计口径只取某些部门角色的数据（例如“信息来源情况”只统计数据管理中心）
- 你保存后没有刷新报表（报表可能有缓存，等一会或刷新）

### 6.2 我不知道某项怎么统计？

- 先看模板字段旁是否有“口径说明/备注”
- 若没有，以你们部门内部口径为准，并在备注（如页面有）说明数据来源（系统、台账、人工统计等）

### 6.3 某项没有发生，是填 0 还是留空？

优先填 **0**（除非页面明确说明允许留空并且留空代表“无数据”）。
