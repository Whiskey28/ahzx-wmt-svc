> 适用于：本地 Docker PG、主程序本地打包后上传 VMware 进行 Docker 构建、U 盘传入内网。**内网环境**：PG 与服务均为 Docker 容器，前端为 Nginx 二进制。可按本文档无脑完成首次与增量投产。

## 三、首次投产（数据库全量 + 应用 + 前端）

### 3.1 外网准备：U 盘物料清单

- `ahc-deploy-bundle.tgz`（内含：docker-compose、nginx.conf、前端静态、镜像 tar）
- `sql-init/` 目录：按顺序执行的 PG 脚本（见下）
- 部署脚本（可选）：`deploy_app_internal.sh`、`deploy_front_internal.sh`、`deploy_db_internal.sh`

### 3.2 数据库（DB 机 192.168.18.112，PG 为 Docker 容器）

**执行顺序**（在已有库 `tdc-credit`、用户 `ahzx` 前提下）：

- 若在 DB 机本机执行，PG 在容器内时可用：`docker exec -i <pg容器名> psql -U ahzx -d tdc-credit < xxx.sql`

1. 信用报送主表：`sql/postgresql/credit-report-2026-02-02-uuid.sql`
2. 固定字段升级：`sql/postgresql/v2/credit-report-2026-02-06-fixed-fields.sql`
3. 长三角征信链：`sql/postgresql/credit-report-2026-02-28-yangtze-credit-chain.sql`
4. 审计触发器：`sql/postgresql/credit-report-2026-02-02-uuid-triggers.sql`
5. 角色/字典：`sql/postgresql/credit-report-role-insert.sql` 等（按需）

```bash
psql -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit -f credit-report-2026-02-02-uuid.sql
psql -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit -f v2/credit-report-2026-02-06-fixed-fields.sql
psql -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit -f credit-report-2026-02-28-yangtze-credit-chain.sql
psql -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit -f credit-report-2026-02-02-uuid-triggers.sql
psql -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit -f credit-report-role-insert.sql
```

### 3.3 应用机（192.168.18.111，应用为 Docker 容器）

1. 拷入 `ahc-deploy-bundle.tgz` 到 `/opt/ahc/`
2. 解压 → `docker load` 导入镜像 → `docker-compose -f docker-compose.yml up -d`
3. 创建 `/data/ahc/logs`，确保 DB/Redis 地址为 `192.168.18.112`

### 3.4 Web 机（192.168.20.65，Nginx 为二进制非容器）

1. 将 bundle 中 `CreditServiceReport/` 拷到 `/opt/nginx/html/CreditServiceReport/`（或实际配置的 root/alias 路径）
2. 将 `nginx.conf` 拷为 `/etc/nginx/conf.d/credit-report.conf`
3. 确认反代后端：`192.168.18.111:48080`
4. `nginx -t && nginx -s reload`

## 四、增量投产（已有大量数据，禁止全量覆盖）

### 4.1 总原则

- **严禁**用 `backup.sql` 全库覆盖
- **严禁**执行含 `DROP TABLE` 的“从零建库”脚本
- 增量分为：**数据库增量**、**Jimu 报表配置更新**、**应用/前端/Nginx 更新**

### 4.2 数据库增量：新增表 / 新增列 / 触发器

仅执行**本次新增或变更**的脚本，不重复执行会 DROP 表的脚本。


| 变更类型  | 脚本示例                                                | 说明                                                     |
| ----- | --------------------------------------------------- | ------------------------------------------------------ |
| 新增表   | `credit-report-2026-02-28-yangtze-credit-chain.sql` | 新建 `report_fill_yangtze_credit_chain`，不 drop 现有表       |
| 新增触发器 | 触发器脚本中为新表补充 `CREATE TRIGGER`                        | 或完整执行 `credit-report-2026-02-02-uuid-triggers.sql`（幂等） |
| 新增列   | 单独的 `ALTER TABLE ADD COLUMN` 脚本                     | 按需编写                                                   |


```bash
# 示例：本次仅新增长三角征信链表
psql -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit -f credit-report-2026-02-28-yangtze-credit-chain.sql
psql -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit -f credit-report-2026-02-02-uuid-triggers.sql
```

### 4.3 Jimu 报表配置更新（仅覆盖 jimu_* 配置表）

**思路**：从源库（本地 Docker / 开发库）导出 jimu 配置表**数据**，在目标库仅清空并导入这些表，不动 `report_fill_`* 等业务数据。

**涉及表**（以 backup 为准）：

- `jimu_dict`、`jimu_dict_item`
- `jimu_report`、`jimu_report_category`
- `jimu_report_data_source`、`jimu_report_db`、`jimu_report_db_field`、`jimu_report_db_param`
- `jimu_report_icon_lib`、`jimu_report_link`、`jimu_report_map`、`jimu_report_share`

**步骤**：

1. （可选）备份目标库现有 jimu 配置，便于回滚：

```bash
pg_dump -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit --data-only --no-owner --no-privileges \
  -t jimu_dict -t jimu_dict_item -t jimu_report -t jimu_report_category \
  -t jimu_report_data_source -t jimu_report_db -t jimu_report_db_field -t jimu_report_db_param \
  -t jimu_report_icon_lib -t jimu_report_link -t jimu_report_map -t jimu_report_share \
  > backup-jimu-$(date +%Y%m%d).sql
```

1. 从源库导出最新 jimu 配置（data-only）：

```bash
pg_dump -h <源库host> -p <源库port> -U <用户名> -d <库名> --data-only --no-owner --no-privileges \
  -t jimu_dict -t jimu_dict_item -t jimu_report -t jimu_report_category \
  -t jimu_report_data_source -t jimu_report_db -t jimu_report_db_field -t jimu_report_db_param \
  -t jimu_report_icon_lib -t jimu_report_link -t jimu_report_map -t jimu_report_share \
  > jimu-config-latest.sql
```

1. 在目标库清空 jimu 配置表并导入：

```sql
-- CASCADE 自动处理外键依赖，RESTART IDENTITY 重置自增
TRUNCATE TABLE
  jimu_dict, jimu_dict_item,
  jimu_report_category, jimu_report,
  jimu_report_data_source, jimu_report_db, jimu_report_db_field, jimu_report_db_param,
  jimu_report_icon_lib, jimu_report_link, jimu_report_map, jimu_report_share
RESTART IDENTITY CASCADE;
```

```bash
psql -h 192.168.18.112 -p 5432 -U ahzx -d tdc-credit -f jimu-config-latest.sql
```

**若仓库有字典增量脚本**（如 `sql/postgresql/jimureport-custom-*.sql`），也可仅执行该脚本做字典级更新，而不做全表 truncate。

### 4.4 应用更新：Docker Compose

1. 本地打包 jar → 上传到 VM → 在 VM 内构建新镜像
2. 导出镜像：`docker save -o ahzx-wmt-svc-new.tar ahzx-wmt-svc:<新版本>`
3. 拷入内网应用机 → `docker load -i ahzx-wmt-svc-new.tar`
4. 在 `ahc-deploy` 目录执行：

```bash
docker-compose -f docker-compose.yml down
docker-compose -f docker-compose.yml up -d
```

1. 检查：`curl http://127.0.0.1:48080/actuator/health`

### 4.5 Nginx 配置更新（二进制安装）

1. 将最新 `nginx.conf` 拷到 Web 机
2. 覆盖：`cp nginx.conf /etc/nginx/conf.d/credit-report.conf`
3. 校验并重载：`nginx -t && nginx -s reload`

### 4.6 前端更新（Nginx 二进制，静态资源直接覆盖）

1. 将最新 `CreditServiceReport/` 拷到 `/opt/nginx/html/CreditServiceReport/`（与 nginx.conf 中 alias 一致）
2. 如需清缓存：`nginx -s reload` 或提醒用户强制刷新

## 七、常见问题


| 问题                   | 处理                                                  |
| -------------------- | --------------------------------------------------- |
| 增量 SQL 里有 DROP TABLE | 不要执行；拆成单独的 CREATE TABLE IF NOT EXISTS 或 ALTER TABLE |
| jimu truncate 报外键错误  | 按依赖顺序 truncate，或使用 `CASCADE`                        |
| 应用连不上 DB             | 检查 profile（ahtest）、DB/Redis 地址是否指向 192.168.18.112   |
| 前端 404 / 白屏          | 检查 basePath、Nginx alias、jmreport/drag/jimubi 是否已反代  |
